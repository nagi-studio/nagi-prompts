# Contributing

每个 Prompt 应该是一个可以独立阅读和复用的案例包。

## 提交一个案例

在 `prompts/<prompt-slug>/` 中提供：

1. `README.md`：说明目标、适用场景、效果截图和使用方式。
2. `prompt.zh-CN.md` 或 `prompt.en.md`：可直接复制的完整 Prompt。
3. `meta.yaml`：标题、类别、语言、状态和预览图。
4. `references.md`：源码是否公开、截图来源、第三方素材与许可边界。
5. 可选的 `assets/<prompt-slug>/`：经过隐私与版权检查的效果截图。

## Prompt 质量要求

- 写清目标、上下文、约束、禁止项和验收标准。
- 不依赖提交者本地文件才能理解。
- 不把私有源码或内部实现伪装成“上下文”。
- Prompt 必须自包含，外部页面不得成为理解或执行它的必要输入。
- 如结果依赖特定模型、工具或日期，明确记录。

## 截图要求

- 只截图公开页面或有权公开的界面。
- 清除账号、邮箱、访客信息、后台数据和访问令牌。
- 优先使用 WebP，保留足够辨认布局的分辨率。
- 文件名描述视口和状态，例如 `desktop-light-home.webp`。
- 在 `references.md` 中说明截图日期、所有者和权利边界。

## License

提交即表示你有权以 CC BY 4.0 公开 Prompt 和文字内容。截图与第三方素材必须单独说明，不得默认纳入仓库许可。
